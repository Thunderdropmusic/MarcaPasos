LOG DE CAMBIOS - PROYECTO ARDUINO MARCAPASOS
===========================================
VERSION: 1.6 (Sistema de Reloj y Persistencia Global)
FECHA: 02-03-2026
-------------------------------------------

[ESTRUCTURA Y LIBRERÍAS]
------------------------
* Inclusión de <TimerOne.h>: Implementación de interrupciones por hardware para un timing preciso.
* Inclusión de "midiPresets.h": Nueva dependencia para la gestión de configuraciones guardadas.
* Organización FULLCODE: Consolidación de todos los módulos en un flujo lineal para evitar errores de enlace.

[SISTEMA DE RELOJ (CLOCK)]
--------------------------
* DUAL CLOCK MODE:
  - Implementación de 'modeMidiClock' (0 = Externo / 1 = Interno).
  - Configuración de Timer1 a 1000ms en el setup para el pulso interno.
  - Función 'clockInterrupt()' vinculada a la interrupción del Timer1 para manejar el avance de los pasos.
* SINCRONIZACIÓN MIDI:
  - Preparación de manejadores para MIDI_CLOCK, MIDI_START y MIDI_STOP.
  - Variable 'enviarPulso' de tipo volatile para asegurar la comunicación correcta entre la interrupción y el bucle principal.

[GESTIÓN DE DATOS GLOBALES]
---------------------------
* SISTEMA DE ESPEJO (Mirroring):
  - Creación de arrays globales (globalNote, globalVelocity, globalOctave, globalMutes) para almacenar el estado del secuenciador de forma centralizada.
  - Variables globales de estado: globalCanal, globalModeSequence, globalEscalaSeleccionada, globalNTotalSteps, etc.
  - Esto prepara el terreno para la carga/guardado de presets sin afectar el motor de audio en tiempo real.

[CAMBIOS EN MIDIPROGRAMMING]
----------------------------
* MOTOR DE TIEMPO:
  - Nuevas variables 'tempo' y 'tempoMicros' para el control de BPM en modo interno.
  - Variable 'tiempoUltimoMidiClock' para calcular el BPM cuando el reloj es externo.
* ESTRUCTURA DE ESCALAS:
  - Se definen las estructuras 'escalasNotas[5][13]' y 'nNotasEscalas[5]', aunque la lógica de mapeo sigue en proceso de integración en el checkPotes.

[INTERFACE Y HARDWARE]
----------------------
* OPTIMIZACIÓN DE TAREAS:
  - Variable 'tareasPesadas' para distribuir la carga de procesamiento y no bloquear el envío de mensajes MIDI NoteOn/Off.
* PINOUT:
  - Se mantiene la configuración de pines de la v1.5, pero con una lectura más integrada dentro del nuevo sistema de tiempos.

-------------------------------------------
NOTAS DEL DESARROLLADOR:
La v1.6 supone el cambio de un secuenciador "esclavo" a uno que puede ser "Maestro" gracias al TimerOne. La infraestructura de variables globales permitirá en la siguiente fase implementar el guardado en memoria de forma transparente para el usuario.
-------------------------------------------
